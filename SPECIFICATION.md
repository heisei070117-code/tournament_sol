# トーナメント表作成アプリ 仕様書（実装プロンプト）

## 1. 目的

ブラウザだけでシングルエリミネーション方式のトーナメントを作成・運営できるWebアプリを実装する。参加者登録、シード設定、組み合わせ抽選、手動配置、試合結果入力、保存、ファイル出力を一つの画面で完結させる。

## 2. 技術構成

- HTML5 / CSS3 / Vanilla JavaScript
- ビルドツール、フレームワーク、外部CDNは使用しない
- サーバー、データベース、ユーザー登録は不要
- 状態は `localStorage` に自動保存する
- ロジックはUIから分離し、Node.js標準テストランナーで単体テスト可能にする

選定理由は、GitHub Pagesを含む静的ホスティングにそのまま配置でき、ローカルでも容易に起動でき、保守対象となる依存パッケージを持たないため。

## 3. 対象範囲

### 対応するもの

- シングルエリミネーション
- 2～256名の任意人数
- 2の累乗でない人数に対するBYE（不戦勝枠）
- 0名から参加人数までの任意シード数
- シード考慮抽選と手動配置
- 各試合の勝者入力と次ラウンドへの反映
- ブラウザへの自動保存
- JSON、SVG、印刷/PDFによる出力

### 対応しないもの

- ダブルエリミネーション、リーグ戦、3位決定戦
- 複数端末間の同期、共同編集、ログイン
- 試合日時・会場・スコアの管理

## 4. 機能要件

### FR-01 大会設定

- 大会名を設定できる。
- 参加者は1行1名のテキスト形式で一括入力できる。
- 空行は無視する。
- 空の名前および大文字小文字を無視した同名参加者の重複をエラーにする。
- 参加者数、トーナメント枠数、BYE数を表示する。

### FR-02 シード設定

- シード数を0から参加人数まで指定できる。
- 各シード番号は一意でなければならない。
- 初期状態では入力順の先頭からシード番号を割り当てる。
- 「第1シード＝チーム名」のように、シード番号を軸とした一覧で現在の割り当てを確認できる。
- 各シード番号のチームをプルダウンから変更できる。
- 選択したチームが別のシードへ割り当て済みの場合、二つの割り当てを自動的に入れ替える。
- 非シードのチームを選択した場合、現在その番号にいるチームを非シードへ変更し、シード総数を維持する。

### FR-03 組み合わせ抽選

- トーナメント枠数は参加人数以上の最小の2の累乗とする。
- シードは標準的な分散順（8枠の場合 `1, 8, 4, 5, 2, 7, 3, 6`）に配置する。
- 非シード参加者はランダムに配置する。
- BYEは可能な範囲で上位シードの対戦相手枠へ優先配置する。
- BYEは1回戦の各試合に最大1枠とし、BYE同士の試合を作らない。
- 参加人数が次の2の累乗を超えた直後の場合、必要最小限の参加者だけが予備戦を行い、それ以外は次ラウンドから出場する公平な構造にする。
- 再抽選できる。再抽選時は既存の試合結果を消去する。

### FR-04 手動配置

- 全トーナメント枠を一覧表示し、各枠の参加者またはBYEを選択できる。
- 既に別枠にいる参加者を選ぶと、二つの枠を自動的に入れ替える。
- 全参加者が重複なく1回ずつ配置されるまで確定できない。
- 同じ1回戦にBYEを2枠配置した状態では確定できない。
- シード番号は参加者の属性として表示するが、手動配置を制限しない。

### FR-05 試合進行

- 各試合に参加者2名を表示する。
- BYEを含む試合は、実在する参加者を自動的に勝者とする。
- 「BYE」と「前試合の勝者が未確定の枠」を内部的に区別し、未確定枠をBYEとして自動勝ち上がりさせない。
- 両者が確定した試合では、参加者名をクリックして勝者を選べる。
- 勝者は次ラウンドへ即時反映する。
- 過去ラウンドの勝者を変更した場合、それに依存する後続試合の結果を消去する。
- 決勝の勝者をチャンピオンとして表示する。
- 組み合わせを維持したまま全結果をクリアできる。
- 次の試合で勝者が確定した時点で、その勝者が進んできた側の接続線だけを赤色かつ太線で表示する。
- 手前の試合に勝者がいるだけでは赤線にせず、次の試合が未決着の場合や次の試合で敗退した側の線はグレーのままにする。

### FR-06 保存と入出力

- 編集のたびに状態を `localStorage` へ保存し、再訪時に復元する。
- 大会情報、参加者、シード、枠、結果をJSONとして保存できる。
- 本アプリが出力したJSONを読み込み、編集状態を復元できる。
- トーナメント表を独立したSVGファイルとして出力できる。
- ブラウザ印刷機能を呼び出し、印刷またはPDF保存できる。

## 5. 非機能要件

- PCを主対象とし、スマートフォンでは設定パネルを縦積みにする。
- 横に長いトーナメント表は水平方向にスクロールできる。
- 各試合から次の試合までを水平線と垂直線で連結し、勝ち上がり先を視覚的に追跡できる。
- 入力エラーは操作箇所の近くまたはトーストで日本語表示する。
- HTMLへ動的に出力する文字列はエスケープし、参加者名によるHTML注入を防止する。
- 外部通信を行わず、入力データを端末外へ送信しない。
- 主要な抽選・シード・勝ち上がりロジックには単体テストを設ける。

## 6. 受入条件

1. 6名・2シードで抽選すると8枠になり、2つのBYEが第1・第2シードに優先配分される。
2. 8名・4シードで抽選すると、第1と第2シードは決勝まで、第1と第4シードは準決勝まで対戦しない配置になる。
3. 5名を手動で8枠へ配置し、3つのBYEを含めて確定できる。
4. 勝者を選択すると次戦へ反映され、前戦の勝者変更時に後続結果がリセットされる。
5. ページを再読み込みしても大会と試合結果が復元される。
6. 保存したJSONを読み込むと同じ編集状態になる。
7. SVG出力ファイルを単独で開いて大会名、各ラウンド、参加者、勝者を確認できる。
8. `npm test` ですべてのテストが成功する。
9. 34名・8シードでは64枠中30試合が「参加者対BYE」、2試合だけが予備戦となり、BYE対BYEや未確定相手による自動勝ち上がりが発生しない。
10. シード割り当て一覧で各シードのチームを一目で確認でき、チーム変更時もシード数と番号の一意性が維持される。
11. 次戦の勝者決定後、その勝者が進んできた側の線だけが赤色になり、敗退側と未決着の線はグレーのままになる。SVG出力にも同じ状態が反映される。
